# Supabase Database Setup Guide

## Quick Setup

### 1. Get Supabase Connection URLs

Go to your Supabase dashboard:
```
https://supabase.com/dashboard/project/YOUR_PROJECT/settings/database
```

You need **TWO** URLs:

#### Connection Pooling URL (for app queries)
```
postgresql://postgres.[project-ref]:[password]@aws-0-[region].pooler.supabase.com:5432/postgres
```
This goes in `DATABASE_URL`

#### Direct Connection URL (for migrations)
```
postgresql://postgres.[project-ref]:[password]@db.[project-ref].supabase.co:5432/postgres
```
This goes in `DIRECT_URL`

### 2. Update `.env.local`

```bash
# Database Configuration (Supabase)
DATABASE_URL="postgresql://postgres.xxxxx:your-password@aws-0-us-west-1.pooler.supabase.com:5432/postgres"
DIRECT_URL="postgresql://postgres.xxxxx:your-password@db.xxxxx.supabase.co:5432/postgres"
```

**Important:**
- Replace `xxxxx` with your project reference
- Replace `your-password` with your actual database password
- The password might need URL encoding if it contains special characters

### 3. Run Prisma Migrations

```bash
# Generate Prisma client
npx prisma generate

# Push schema to Supabase (creates tables)
npx prisma db push

# Or run migrations properly
npx prisma migrate deploy
```

---

## Why Two URLs?

### `DATABASE_URL` (Connection Pooling)
- **Use for:** Application queries (SELECT, INSERT, UPDATE)
- **Port:** 5432 (pooler)
- **Benefits:** Handles many concurrent connections efficiently
- **Limits:** Cannot run migrations or schema changes

### `DIRECT_URL` (Direct Connection)
- **Use for:** Migrations and schema changes
- **Port:** 5432 (direct)
- **Benefits:** Full PostgreSQL access
- **Limits:** Limited concurrent connections

## Troubleshooting

### Error: "Authentication failed"
```
P1000: Authentication failed against database server
```

**Solutions:**
1. Check password is correct
2. URL-encode special characters in password:
   ```typescript
   // If password is: p@ssw0rd!
   // Encode as:      p%40ssw0rd%21
   ```
3. Verify project reference ID is correct
4. Check database is not paused (Supabase pauses inactive DBs)

### Error: "Database does not exist"
```
P1003: Database 'postgres' does not exist
```

**Solution:** Supabase always uses database name `postgres`, not `limbo_wallets`

### Error: "Too many connections"
```
P1001: Can't reach database server
```

**Solution:** Use connection pooling URL (`DATABASE_URL`), not direct URL for app

## Prisma Schema Configuration

Your `prisma/schema.prisma` should have:

```prisma
datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")      // Connection pooling
  directUrl = env("DIRECT_URL")        // Direct connection for migrations
}
```

## Migration Commands

### Push schema (quick, for development)
```bash
npx prisma db push
```
- No migration files created
- Directly syncs schema to DB
- Fast for development

### Create migration (proper, for production)
```bash
npx prisma migrate dev --name init
```
- Creates migration files
- Tracks schema changes
- Better for version control

### Deploy migrations (production)
```bash
npx prisma migrate deploy
```
- Runs pending migrations
- Use in production/CI/CD

### Reset database (DANGER!)
```bash
npx prisma migrate reset
```
- Drops all tables
- Re-runs all migrations
- **Use with caution!**

## Current Schema

Your app uses these tables:

### `users`
- Stores Farcaster user profiles
- Primary key: `id` (CUID)
- Indexed on: `farcaster_id`

### `wallets`
- Stores encrypted server-side wallets
- Primary key: `userId` (references users.id)
- Indexed on: `address`, `lastUsed`, `createdAt`

### `wallet_transactions`
- Optional transaction history
- Primary key: `id` (auto-increment)
- Foreign key: `userId` → `wallets.userId`

- ✅ `prisma/schema.prisma` - Source of truth

## Common Issues

### Issue: "Relation does not exist"
**Cause:** Tables not created
**Fix:** Run `npx prisma db push`

### Issue: "Unknown field" error in code
**Cause:** Prisma client out of sync
**Fix:** Run `npx prisma generate`

### Issue: "Migration failed"
**Cause:** Using pooling URL for migrations
**Fix:** Ensure `DIRECT_URL` is set and correct

### Issue: "Connection timeout"
**Cause:** Supabase database paused
**Fix:** Go to dashboard and unpause database

## Security Notes

### Never commit:
- ❌ `.env.local` (has passwords)
- ❌ Database passwords
- ❌ Connection strings

### Do commit:
- ✅ `.env.example` (template)
- ✅ `prisma/schema.prisma`
- ✅ `prisma/migrations/*` (if using migrations)

### Rotate passwords if exposed:
1. Go to Supabase Settings → Database → Database password
2. Click "Reset database password"
3. Update `.env.local` with new password
4. Restart your app
