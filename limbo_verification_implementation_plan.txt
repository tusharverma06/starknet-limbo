# 🧩 Limbo Game - Verifiable Bet & Randomness System (Next.js Backend)

## 🎯 Goal
Enable players to see and verify that every bet’s outcome, randomness, and payout are **provably fair** and **cryptographically verifiable**.

---

## 🧠 Verification Steps (with Implementation Plan)

### 1. Verify Login Message Signature
- **What:** Ensure the player’s login/auth key is genuinely signed by their custodial wallet.
- **How:**
  - Backend: Verify wallet signature with `ethers.utils.verifyMessage()` and store the `authKey`.
  - Frontend: Display “✅ Wallet Authenticated”.

### 2. Verify Bet Request Signature
- **What:** Confirm each bet request (amount, choice, etc.) was signed with the user’s key — not faked.
- **How:**
  - Backend: Sign bet request hash with user’s or server’s verifiable key.
  - Frontend: Display “🪙 Bet Signed & Recorded”.

### 3. Verify Randomness Proof
- **What:** Check that the random seed used to decide the result matches the **commit-reveal hash** or oracle value.
- **How:**
  - Backend: Publish `serverSeedHash` before the bet; reveal `serverSeed` after the result.
  - Frontend: Show both seeds and a “🔍 Verify Randomness” button to recheck with SHA256(serverSeed).

### 4. Verify Payout
- **What:** Ensure payout math matches the game’s logic.
- **How:**
  - Backend: Log bet inputs + result + computed payout.
  - Frontend: Show “💰 Payout Verified: X.XX BASE”.

### 5. Verify Shard Reward
- **What:** Validate bonus or shard reward calculations.
- **How:**
  - Backend: Emit logs or signed proofs for shard allocation.
  - Frontend: Show shard proof hash.

### 6. Verify Settlement
- **What:** Confirm bet settlement (winner, amount, timestamp) matches calculations.
- **How:**
  - Backend: Save computed settlement values + sign them.
  - Frontend: Display signed JSON proof in results modal.

### 7. Match On-chain Transaction
- **What:** Ensure payout transaction on Base chain matches backend records.
- **How:**
  - Backend: Record the transaction hash.
  - Frontend: Show “🔗 View on BaseScan” link with transaction hash.

---

## ⚙️ Next.js Backend Tasks

1. **Generate Server Seed per Round**
   ```js
   const serverSeed = crypto.randomBytes(32).toString("hex");
   const hash = sha256(serverSeed);
   ```
   Store `hash` before the game starts.

2. **Accept Bet**
   - Record user input, timestamp, and auth signature.

3. **Reveal Round**
   - Reveal `serverSeed` after result.
   - Combine with `clientSeed` → `finalRandom = sha256(serverSeed + clientSeed)`.

4. **Compute Result + Payout**
   - Apply game logic.
   - Sign result with backend’s verifiable key.

5. **Publish Proof JSON**
   ```json
   {
     "serverSeed": "...",
     "clientSeed": "...",
     "hash": "...",
     "random": "...",
     "result": "...",
     "payout": "...",
     "signature": "..."
   }
   ```

6. **Serve Proof via `/api/verify/[betId]`**
   - Users or frontend can fetch and verify independently.

## 📖 Glossary (Simple)

| Term | Meaning |
|------|----------|
| **Commitment** | Publishing a hash of a secret before revealing it to prove honesty later. |
| **Reveal** | Showing the secret (serverSeed) that matches the earlier hash. |
| **Auth Key** | A signed message that proves the user’s wallet identity. |
| **Server Seed** | Secret number generated by backend before the bet. |
| **Client Seed** | Random value from the player to mix randomness. |
| **Final Random** | SHA256(serverSeed + clientSeed). |
| **Proof JSON** | Data showing all cryptographic evidence for fairness. |
| **Verification Modal** | UI section showing all check results. |
| **BaseScan Link** | URL showing the blockchain transaction. |
